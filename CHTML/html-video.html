Dive into Cinematics with <video> Element

The <video> element is used to embed video files in a web page. It allows you to include video content and provides a standardized way to present videos on the web.

The <video> element supports various video formats, and it includes built-in controls for the user to play, pause, adjust volume, and perform other actions.

Here are some common attributes associated with the <video> element:

  • src: The src attribute is required and specifies the path or URL to the video file
  • controls: The controls attribute adds playback controls to the video player, allowing users to play, pause, adjust volume, and more.

    Video by Danil_Shostak from Pixabay

  • width and height: These attributes control the dimensions of the video player. Similar to the <img> element, they allow you to set the width and height of the video display.
  • autoplay: The autoplay attribute, when present, makes the video start playing automatically as soon as it is loaded. However, note that some browsers have restrictions on autoplay to prevent annoying user experiences, especially with sound.
  • loop: The loop attribute, when present, causes the video to loop back to the beginning and restart when it reaches the end.

  • muted: The muted attribute, when present, mutes the audio of the video by default. This is often used in combination with autoplay to create a muted, autoplaying video.
  • poster: The poster attribute specifies an image that is shown while the video is downloading, or until the user hits the play button. It is a good practice to provide a poster for better user experience.

These attributes give you control over how videos are presented and played on your web pages, providing an interactive and engaging multimedia experience for users.

fallback content

you can provide fallback content within the <video> element to be displayed if the browser does not support the video tag or if none of the specified video formats can be played.

The fallback content is placed between the opening <video> tag and the <source> elements.

This content is shown to users who have browsers that do not support the <video> tag or cannot play any of the specified video formats.

The <source> elements provide different video sources in different formats, allowing browsers to choose the most suitable format based on compatibility.

If the browser supports the <video> tag and can play one of the specified video formats, it will use the video content. If not, it will display the fallback content.

This practice ensures a better user experience, as users who cannot view the video will still receive some information or alternative content. Additionally, you can style the fallback content to make it visually appealing and informative.

The <video> element is a powerful tool for incorporating multimedia content into web pages, providing a standardized and user-friendly way to present videos. It supports a variety of video formats, making it versatile for different browsers and devices.
